网络术语

bandwidth 带宽， Ethernet 以太网，是局域网最常见的组网方式。最简单的以太网就是多个主机通过一根电缆连接在一起。 复杂一些的以太网则通过双绞线、无线、集线器等设备连接在一起。以太网中报文碰撞现象的发生比较频繁，繁忙时可能使带宽下降至 30% 。 IP 互联网协议（ Internet Protocol ）的缩写。 LAN 局域网（ Local Area Networks ）的缩写。 LAN 是网络的连接层（ Link Layer ），也是逻辑意义上的物理层。 packet switching 数据包交换，是多个用户共享同一个网络连接的解决方案。 TCP/IP 是一种实现数据包交换的协议（栈）。 Packets 数据包， TCP 传输控制协议（ Transmission Control Protocol ）的缩写。 throughput 吞吐， WAF Web 应用防火墙（ Web Application Firewall ）。 WAF 部署在 Web Application 的前端，主要用于防御针对网络应用层的攻击，例如， SQL 注入，跨站脚本攻击，参数篡改、应用平台漏洞攻击、拒绝服务攻击等。

AWS Lambda

简介 AWS Lambda 是 AWS 提供的一种计算服务，服务内容包括： 提供高可用性计算基础设施 提供计算资源的自动管理任务，包括服务器和操作系统维护、容量配置和自动伸缩、代码监控和日志记录等 仅在需要时运行 Lambda Function ，并根据计算时间计费 Lambda Function 是部署在 Lambda 服务上的程序包，仅在需要时运行，并根据计算时间计费，这项服务可以极大地降低应用在云平台上的使用成本，事实上，这也是 Lambda 服务提供的核心价值之一。 使用 Lambda 服务需要引入一些新的编程思想，例如， Lambda Function 应该尽可能做到原子化和无状态计算。 原子化意味着高度内聚。原子化的本质是避免对外部服务的依赖。对外部服务过多的依赖通常是使应用系统趋于复杂的重要因素。例如，当同步依赖的外部服务需要长时间运行， Lambda Function 无法快速销毁，这可能破坏 Lambda 服务为应用带来的好处。无状态计算则是指在计算期间不涉及中间数据和计算任务状态的存储，这使 Lambda Function 简单而纯粹，并使其应用场景更加广泛。例如，可配置无状态原子计算引擎。 AWS Lambda 提供的主要功能 并发和伸缩控制 Concurrency and scaling controls 术语 术语 描述 layer 一个层是一个 .zip 文件（不适用于镜像），使用层技术，可以使部署包尽可能地小，因为部署包依赖的库可以通过一个新的层提供 runtime 在 Lambda 运行环境中工作的一个编程语言相关的环境，负责转发调用事件，上下文信息，以及最终的响应 handler 函数代码中由 Lambda 调用并开始执行的方法 code signing 代码签名 execution role 授权 Lambda 函数去访问 AWS 服务和资源的 IAM 角色（ AWS Identity and Access Management (IAM) role ） unpublished version 未发布的版本，当函数更新时，会发生相应的变化 published versions 已发布的版本，是函数的一个快照，不会发生变化 aliases 函数别名，可以设置使之对应于一个版本 ARN Amazon Resource Name ， 例如，execution role ARN 的样式为 arn:aws:iam::your-account-id:role/execution_role 常用命令 AWS CLI 卸载 AWS CLI version 1
