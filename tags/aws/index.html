AWS Lambda

Lambda 运行环境 Lambda 运行环境生命周期 三个阶段： Init 第一次调用时触发（或根据并行预配置）。基本过程是，基于配置创建运行环境，下载函数及各层代码，初始化扩展和运行时，运行函数初始化代码。 Invoke Lambda 调用函数 Handler。 Shutdown 一段时间没有收到任何调用后触发。 Lambda 函数发布 部署包有两种形式： 容器镜像 （ container image ） .zip 文件 容器镜像 一般使用 Docker CLI 把函数代码和相关依赖打包成一个镜像，也可以使用 AWS Serverless Application Model （ AWS SAM ）来创建。 AWS 提供了一些开源的基础镜像，这些基础镜像包含了相关编程语言的运行时，以及支持镜像在 AWS Lambda 中运行的组件，例如，运行时接口客户端 （ runtime interface clients ），用于管理 Lambda 和函数代码之间的交互。 镜像创建后，使用 AWS CLI 上传至 Amazon Elastic Container Registry （ Amazon ECR ）。
